Summary

goktech-agent 是面向 AI 实训营、FDE 现场交付、客户试点 MVP 和团队资产沉淀的智能体工作平台。它是帮助 Echo、Delta、数据、知识、BU 等角色协同完成 AI 项目从调研、原型、试点、移交、交付、沉淀以及复用的工作平台。

本 PRD 基于《GW-AI行动计划》第七章整理，重点围绕五条产品主线：

📊

👥
团队共享
空间共享与资产复用
🎨
AI UI 人机交互
结构化输出与反馈闭环

Background

3.1 当前背景

正在推进 GW AI 行动方案。行动方案要求形成"建体系、搭团队、抓机会、做示范、沉淀资产"的闭环。FDE 团队需要在客户现场快速理解业务、接入数据、构建 Agent 原型、验证 MVP，并把项目经验沉淀为可复用资产。

痛点：项目材料、数据资产、Agent 配置、工具能力、知识包、评测集和 SOP 容易分散在个人手中。团队协同、权限管理、资产复用和客户演示也缺少统一平台支撑。

3.2 为什么现在建设

AI 实训营、FDE 现场交付和 2026 年首批试点都需要一个统一工作平台。没有平台，团队会反复处理文件、数据、知识库、提示词、工具和权限问题，影响 MVP 交付速度，也难以形成可复制的交付包。

3.3 产品机会

goktech-agent 可以成为AI CoE 的核心工作平台。它把数据工作台、多智能体协同、技能插件、团队空间和 AI UI 汇聚在一起：

  • 支撑 FDE 做 0 到 1
  • 支撑 BU 做 1 到 N
  • 支撑 AI CoE 沉淀复用资产

Objective

4.1 产品目标

建设一个面向 AI 项目交付的智能体工作平台，让 FDE 小队可以快速接入客户资料和业务数据，构建 Agent 原型，协同推进试点，并把成功经验沉淀为团队可复用资产。

4.2 业务目标

  • 缩短 FDE 小队从业务材料到可演示 Agent 原型的时间
  • 提升数据、知识、Agent、工具和交付模板的复用率
  • 降低团队协同时的信息丢失和重复劳动
  • 支撑 AI 实训营、客户试点 MVP 和 BU 移交
  • 形成面向 GW AI 项目的平台化交付能力

Market Segment(s)

5.1 主要用户

用户群体 核心任务 主要痛点
FDE Echo 访谈客户、识别机会、管理机会卡、做价值复盘 业务材料分散，机会判断和客户反馈难沉淀
FDE Delta 快速构建 Agent 原型、工具调用和演示 Demo 工具、技能、提示词和插件难查找，原型能力难复用
数据角色 接入数据、做 ETL、建 RAG、做知识图谱和本体 数据文件、结构化表、权限、脱敏和质量检查缺少统一工作台
知识角色 建知识包、标签体系、评测样本和运营看板 知识切片、审核、质量评测和更新机制不统一
BU 交付团队 接收 FDE 移交包，做规模化交付和运维 MVP 经验难变成标准交付包，配置和 SOP 难复用
客户/业务用户 试用 Agent，看结果，给反馈 AI 过程不可见，结果难理解，反馈入口不清晰
AI CoE 建平台能力、方法论和复用资产 资产沉淀缺少统一载体，复用效果难统计

5.2 使用场景约束

  • 安全优先 平台要支持客户现场交付，必须重视权限、空间隔离、数据脱敏和审计日志
  • 快速验证 平台要服务 FDE 快速验证，首版不应追求大而全，应优先支持高频、刚需、可验收的能力
  • 资产沉淀 平台要支撑团队复用，所有重要产出都应能沉淀为项目资产
  • 用户体验 平台要支撑非技术用户理解结果，AI UI 要能展示文件、表格、看板、流程和任务状态

Solution

7.1 UX / Prototypes

首版产品建议采用"工作空间 + 对话任务 + 数据资产 + Agent/技能 + 项目资产库"的主结构。

7.1.1 核心页面

页面 目标 关键能力
首页 / 工作台 让用户看到当前项目、任务和最近资产 最近文件、最近任务、待处理事项、常用 Agent、常用技能
个人空间 管理个人文件和资料 上传、下载、删除、建文件夹、拖拽引用、首次对话引用文件
团队空间 管理团队项目资料 团队成员、共享文件夹、权限、项目资料沉淀
数据工作台 原始数据的摄取、清洗和结构化存储 文档解析（MinerU）、ETL 任务、字段映射、去重脱敏、Supabase 表管理、任务日志
知识工作台 构建 Agent 可理解的知识表示 RAG 知识库、RAG 评测面板、知识图谱（实体/关系/标签/查询）、本体工作台
Agent 中心 发现、管理和评测单个 Agent 智能体分类、业务场景分类、团队归属、权限管理、Agent 评测入口
Agent 团队 创建和管理多智能体编排 团队创建、成员 Agent 分配、编排模式（串行 / 并行 / 路由）、团队测试、版本管理
技能/插件中心 发现和运行工具能力（消费端） command 搜索、技能分类、插件搜索、权限说明
工具接入管理 注册和配置平台可用的外部连接（管理端） MCP Server 注册、HTTP API 接入、连接可用性测试、按团队/角色授权
多任务对话 FDE 和团队成员同时推进多个 AI 任务 上下文隔离、任务状态、历史记录、单/多 Agent 对话、结构化输出、文件拖拽引用
Agent 业务视图 面向业务用户（非 FDE）的 Agent 前台界面 表格/看板/流程图渲染、表单驱动对话、推理过程摘要、引用文件预览、反馈入口
项目资产库 沉淀和复用团队发布的只读资产快照 快照版本列表、跨团队浏览搜索、一键复用（Fork）、资产被引用次数统计
反馈与改进任务台 汇聚业务用户反馈与试点问题 反馈/Bug/需求列表、任务分配与状态流转、反馈回链、改进闭环跟踪
权限与成员管理 平台权限治理的管理端入口 用户白名单、团队与成员管理、空间隔离配置、访问授权、审计日志查询
运营统计后台 查看平台使用、效果与复用成效 用户、Agent、skills、plugins、token、任务成功率统计；资产复用率排行

7.1.2 关键用户流程

流程一：FDE 快速构建 MVP

1
Echo 在团队空间上传客户材料

将客户提供的原始材料上传到团队共享空间

2
数据角色在数据工作台完成数据处理

文件解析、结构化数据导入和脱敏

3
知识角色创建知识库

RAG 知识库、标签体系和评测样本

4
Delta 在 Agent 中心构建 Agent

选择或创建 Agent，绑定知识库、工具和提示词

5
小队在多任务对话中调试 Agent

记录问题和评测结果

6
试点通过后，将资产保存到项目资产库

提示词、配置、评测集和 SOP 沉淀

流程二：团队共享和 BU 移交

1
FDE 小队在团队空间整理 MVP 产出

形成标准交付文件夹结构

2
FDE 负责人在团队空间执行"发布"操作

填写版本号和发布说明，生成只读快照

3
快照进入项目资产库

BU 成员和 AI CoE 可跨团队发现和查阅

4
团队空间内容继续可编辑

不影响已发布的快照版本

5
BU 根据快照接手后续交付和运营

使用部署清单和运维 SLA

流程三：业务用户试用和反馈

1
业务用户进入指定 Agent 或业务界面
2
用户查看文件引用、推理过程、表格、看板或任务状态
3
用户复制结果或提交反馈
4
反馈进入「反馈与改进任务台」

转为可分配、可跟踪的改进任务

5
FDE 小队领取任务并更新对应配置

回链更新知识、提示词、工具或 Agent 配置

流程四：跨团队资产复用

1
新项目小队在项目资产库检索已发布快照

按行业/场景/标签搜索

2
小队预览快照内容

文件、Agent 配置、提示词、评测集、SOP

3
对目标快照执行"一键复用（Fork）"

将所选资产复制进自己的团队空间

4
小队在复制基础上做客户定制

无需从零搭建

5
平台记录该快照被复用次数

反映到运营统计后台的复用率指标

反馈入口 提供反馈建议入口，沉淀新需求和 Bug 必须支持

Agent 业务视图（业务用户侧）

功能 描述 首版范围
结构化输出渲染 Agent 输出渲染为表格、看板、流程图或任务状态 表格和任务状态
表单驱动对话 Agent 可生成结构化表单（字段、选项、日期）供业务用户填写后继续处理 后续版本
推理过程摘要 可折叠展示 Agent 的推理步骤和引用来源 基础引用来源
引用文件卡片预览 对话中引用的文件以卡片形式展示，支持内联预览 必须支持
结果复制与导出 支持复制对话内容、导出表格为 CSV 或 Excel 复制（导出后续）
反馈与评分 支持点赞/踩、评论和修改请求，反馈进入 FDE 改进任务列表 必须支持

7.2.7 反馈与改进闭环

功能 描述 首版范围
反馈与改进任务台 汇聚业务反馈、Bug 与需求，转为可分配、可跟踪的改进任务 必须支持
反馈回链 任务回链到对应 Agent / 知识库 / 提示词，闭环跟踪改进 必须支持

7.3.2 非功能要求

类别 要求
🛡️ 安全 客户资料必须按空间和团队隔离；敏感数据需要脱敏；关键操作保留审计日志
⚡ 稳定性 大文件上传、JSON buffer、临时文件清理等已知问题必须纳入治理
👥 可用性 非技术用户应能完成文件上传、Agent 试用、结果复制和反馈提交
♻️ 可复用性 重要配置、提示词、评测集、SOP 和交付包应能被复制到新项目

🔄 可扩展性 预留企微协同、GraphRAG、多 Agent 编排和动态业务工具扩展空间

7.4 Assumptions

假设 风险 验证方式
FDE 小队会把 goktech-agent 作为主要工作平台 如果平台体验不好，团队仍会回到个人文件和聊天工具 在 AI 实训营中试用，观察真实使用频率
Supabase 能满足首版结构化数据管理需求 如果权限、部署或性能不满足，需要更换方案 用 1-2 个真实试点数据表验证
首版 RAG 能支撑知识运营和教培类 MVP 如果文档质量差或切片策略不合适，效果会不稳定 建立评测样本，比较人工基线和检索效果
团队愿意沉淀项目资产 如果资产沉淀流程太重，会被跳过 将沉淀动作嵌入交付流程和验收标准
客户和业务用户需要可视化 AI UI 如果界面复杂，会增加使用成本 用试点用户测试文件预览、表格、任务状态和反馈入口
权限和空间隔离是客户资料进入平台的前提 如果安全边界不清，客户资料不能进入平台 在首版验收中优先验证白名单、团队权限和审计日志

Release

8.1 版本策略

产品建设建议分三步推进：先支撑 AI 实训营和内部 MVP，再支撑 FDE 现场试点，最后支撑团队资产复用和 BU 规模交付。

V0.1：AI 实训营和内部速赢版
建议周期：2-4 周

目标：让 FDE 小队能在训练营和内部演示中完成文件管理、基础 RAG、Agent 原型和项目资产沉淀。

范围：

  • 个人空间和团队空间基础文件管理
  • 文件上传、下载、删除、建文件夹、批量导入
  • Markdown 和图片包下载
  • 基础 RAG：文档解析、切片、检索、引用溯源
  • Agent 分类和基础对话
  • /command 搜索、技能分类和插件搜索
  • 多任务对话的基础能力
  • 团队空间标准文件夹结构规范
  • 文件预览、对话复制、会话名称、反馈入口

不包含：

  • 完整企微协同
  • 完整知识图谱和 GraphRAG
  • 复杂多 Agent 自动编排
  • 完整运营统计后台
V0.2：FDE 现场试点版
建议周期：4-8 周

目标：支撑 1-2 个真实 FDE MVP，完成数据接入、权限治理、团队协作和评测闭环。

范围：

  • Supabase 结构化数据后端（表、字段、权限和查询管理）
  • ETL 导入、清洗、字段映射、去重、脱敏和任务日志
  • RAG 评测面板、知识图谱和本体基础工作台
  • 团队对话和项目上下文沉淀
  • 团队空间资产发布（Release）+ 项目资产库
  • 用户白名单、团队权限、Agent 权限、技能权限和空间隔离
  • 基础审计日志 + 运营统计
  • Agent 输出表格和任务状态
  • 工具接入管理：MCP Server / HTTP API 注册与连通性测试
  • 多智能体团队基础编排（串行 / 并行）
  • 项目资产库一键复用（Fork）和资产复用率统计
  • 反馈与改进任务台：业务反馈转改进任务并回链 Agent
V1.0：项目资产沉淀和 BU 移交版
建议周期：8-12 周

目标：让成功 MVP 能转化为标准交付包，支撑 BU 复制推广和持续运营。

范围：

  • 项目资产库增强：快照标签、行业分类和跨团队推荐
  • 快照对比：查看同一团队空间不同版本之间的变化
  • 标准交付包管理、Agent 配置复制和迁移
  • 评测集、SOP、运营看板和部署清单沉淀
  • 定时任务：资料同步、报告生成、知识库更新、运营日报
  • 更完整的业务界面：表格、看板、流程、任务状态
  • 动态业务工具开发和 FDE 工具箱沉淀
  • 企微消息渠道和群聊 Agent 试点

8.5 后续版本方向

  • GraphRAG 深化和跨知识库推理
  • 多 Agent 编排和团队协作自动化
  • 更细粒度的数据权限和客户侧部署模式
  • 更完善的平台运营分析和成本分析
  • 面向不同业务线的标准 Agent 模板库

MVP 验收标准

验收项 标准
📁 文件管理 用户能在个人空间和团队空间完成文件上传、下载、删除、建文件夹和批量导入
🗄️ 数据接入 用户能导入至少一种结构化数据，并完成字段查看和基础查询
📚 RAG 用户能基于上传文档创建知识库，并查看引用来源
🤖 Agent 原型 用户能选择或创建 Agent，并绑定文件、知识库或技能完成对话任务
⚡ 多任务 用户能同时运行多个任务，并查看任务状态和历史记录
🔧 技能/插件 用户能通过搜索找到 command、skills 或 plugins
👥 团队协作 团队成员能共享文件、对话结论和项目资产
🔐 权限治理 非授权用户不能访问受限空间、文件、Agent 或技能
📦 资产发布 试点结束后能从团队空间发布至少一个只读快照（Release），快照含文件、Agent 配置、提示词、评测集和 SOP，且跨团队可查阅
🎨 AI UI 用户能预览文件、复制结果、查看结构化输出并提交反馈
🔌 工具接入 管理员能注册 MCP Server 或 HTTP API 并完成连通性测试，按团队/角色授权
👥 多智能体团队 用户能创建 Agent 团队、分配成员 Agent 并以串行/并行模式跑通一次任务
♻️ 资产复用 用户能从项目资产库一键复用（Fork）一个快照到团队空间，平台记录复用次数
🔄 反馈闭环 业务反馈能进入反馈与改进任务台，分配并回链到对应 Agent / 知识库 / 提示词
⚙️ 权限管理 管理员能在权限与成员管理页配置白名单、团队/Agent/技能权限并查询审计日志
