## 一、文档说明

### 1.1 用例编号规则

```
TC-{模块缩写}-{三位序号}
```

| 模块 | 缩写 | 模块 | 缩写 |
|------|------|------|------|
| 认证与用户体系 | AUTH | 营销地图 | MMAP |
| 组织架构 | ORG | 拜访记录 | VISIT |
| 客户管理 | CUST | 对话与 Skill/Plugin | CHAT / SKILL |
| 项目管理 | PROJ | 候选区/采纳/审批 | CAND |
| 权限模型 | PERM | 知识归档 | ARCH |
| 业务地图 | BMAP | 核心业务闭环（端到端） | FLOW |
| 导航与项目选择器 | NAV | 非功能测试 | NON |

### 1.2 优先级定义

| 优先级 | 含义 | 适用范围 |
|--------|------|---------|
| **P0** | 阻塞级 / 冒烟 | 核心主流程、安全防线、权限隔离、数据一致性，必须通过 |
| **P1** | 重要 | 重要业务功能、关键异常分支、主要边界 |
| **P2** | 一般 | 辅助功能、次要边界、体验优化 |
| **P3** | 低 | 极端边界、长流程、性能压测 |

### 1.3 测试类型定义

| 类型 | 说明 |
|------|------|
| 功能 | 正常业务功能验证 |
| 边界 | 边界值、极值、临界条件 |
| 异常 | 非法输入、错误流程、冲突场景 |
| 权限 | 角色权限、项目隔离、越权访问 |
| 接口 | 后端 API 入参/出参/状态码校验 |
| 流程 | 端到端业务闭环 |
| 数据 | 数据一致性、并发、事务 |
| UI | 前端展示与交互 |
| 性能 | 响应时间、吞吐、渲染 |
| 安全 | 注入、越权、敏感信息泄露 |
| 兼容 | 浏览器、文件格式、响应式 |

### 1.4 用例格式约定

每条用例以表格行呈现，列含义：

| 列 | 说明 |
|----|------|
| 用例编号 | `TC-XXX-NNN` |
| 用例标题 | 一句话描述被测点 |
| 优先级/类型 | `P0/功能` 等 |
| 前置条件 | 执行前必须满足的状态、数据 |
| 测试步骤 | 编号步骤（① ② ③ 或 1) 2) 3)） |
| 预期结果 | 对应每个步骤的预期，含状态码 / 数据状态 / UI 反馈 |

> 测试步骤中的"角色账号"约定：`admin`=系统管理员；`owner`=项目主手；`deputy`=项目副手；`user`=普通用户；`userB`=另一非成员用户。
