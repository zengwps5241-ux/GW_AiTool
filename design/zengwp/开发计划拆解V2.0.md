---
如何用 Claude Code 执行本计划

每张 `CC-XX` 任务卡都是**可直接喂给 Claude Code 的 prompt 骨架**。推荐工作流：

1. **按依赖顺序逐卡执行**，不要跳序。
2. **每卡执行步骤**：把卡的"Claude 输入 + 目标文件 + 验收"贴给 Claude Code → Claude 实现 → 跑"验证命令" → 人工对照"验收"勾掉。
3. **每卡完成后提交一次**（遵循 `feat:/fix:` 规范），保持可回滚。
4. **到达里程碑门禁**（第十二章）才进下一 Phase。

> 任务卡字段说明：
> - **Claude 输入**：Claude 实现前需要读的基准文档章节 / 参考代码
> - **目标文件**：Claude 将创建或修改的具体路径
> - **验收**：可勾选的完成标准
> - **验证命令**：客观判定通过的命令（`pytest` / `npm run build` / 手动跑）

## 总览：6 个 Phase，约 11-16 周

| Phase | 名称 | 周期 | 核心交付 |
|-------|------|------|---------|
| P1 | 地基重构 | 2-3 周 | 认证体系 + 组织架构 + 客户/项目模型 + 权限中间件 |
| P2 | 数据模型与 API | 2-3 周 | 全部业务实体 + CRUD API + 草稿/采纳/版本机制 |

| P5 | 联调与打磨 | 1-2 周 | 端到端联调 + 审批流程 + 版本回溯 + 角色关系图 |
| P6 | 系统设置 | 1-2 周 | 角色管理 + 菜单管理 + 审计日志 + 用户管理增强 + 侧边栏动态渲染 |
