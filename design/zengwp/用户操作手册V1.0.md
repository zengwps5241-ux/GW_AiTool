## 一、系统概述

单击左侧导航栏（Sidebar）的各菜单进入对应页面。系统共分四个业务页面与三个辅助分区：

```
Sidebar
├── 💬 对话              ← AI 交互入口，所有工作流在此触发
├── 🗺️ 业务地图          ← 查看/编辑假设与现状地图
├── 🎯 营销地图          ← 查看/编辑角色卡
├── 📝 拜访记录          ← 查看拜访时间线与证据
├── 📁 个人空间          ← 私有文件，按项目自动归档
├── 👥 团队空间          ← 公开资产与方法论库
└── ⚙️ 设置（仅 admin）  ← 用户/组织/触发词/种子数据
```

> 主线：**对话页"生产"数据 → 采纳后进入业务地图/营销地图/拜访记录"沉淀" → 个人空间/团队空间"归档"**。

## 二、登录与注册

系统采用自建账号体系（手机号 / 用户名 + 密码）。

### 2.1 注册账号

单击登录页的"注册账号"，填写手机号（或用户名）、密码及基本信息后提交。提交后账号状态为"待审批"，需等待管理员审批通过后方可登录。

### 2.2 登录

单击登录页输入手机号 / 用户名与密码，单击"登录"进入系统。

### 2.3 登出

单击右上角头像，选择"登出"，会话随即失效并返回登录页。

> 账号也可由管理员在后台直接创建，此类账号默认为"启用"状态，无需审批。

## 三、对话

对话是系统唯一的 AI 交互入口。所有工作流（生成地图、整理纪要、验证假设等）都在此触发、产出、确认。

### 页面总览

```
┌─────────────────────────────────────────────────────┐
│ 顶部栏：项目选择器 + 阶段标签 + 项目成员              │
├──────────┬──────────────────────────┬───────────────┤
│ 会话列表  │ 消息流                    │ Banner区      │
│ (倒序)    │  · 我的消息               │ · 待审核提醒  │
│ +新建对话 │  · AI回复(思考/证据/候选) │ · 消歧确认    │
│           ├──────────────────────────┤              │
│           │ 输入区：WF chip + 文本框  │              │
│           │           + 文件拖入 + 发送│              │
└──────────┴──────────────────────────┴───────────────┘
```

### 3.1 顶部栏

**项目选择器**：位于顶部栏左侧，下拉选择"客户 → 项目"。切换项目即切换整个数据上下文，业务地图、营销地图、拜访记录共享同一选择。发起任何操作前请先确认选中的是目标项目。

**项目阶段标签**：显示当前项目所处阶段（线索筛查 / 拜访准备 / 现场验证 / 复盘）。

**项目成员**：显示当前项目的 Owner 与 Deputy 成员。

### 3.2 会话管理

**新建对话**：单击左侧"新建对话"按钮，创建一条属于当前项目的新会话，系统自动关联该项目专属 Agent（已绑定全部 Skill / Plugin）。

**切换会话**：单击左侧会话列表中的任一历史会话，即可切换查看并继续。

### 3.3 工作流触发（底部输入区 chip）

底部输入区提供 5 个工作流按钮，单击即直接触发对应工作流，无需意图识别：

**📋 生成假设地图（WF07）**：单击后 AI 分步生成业务假设地图 L1→L2→L3→L4，过程中自动搜索公开信息。建议先上传部分客户资料后再使用。

**📝 生成拜访前方案（WF06）**：单击后 AI 输出拜访目标、沟通要点、访谈问题、资料清单。无前置依赖，随时可用。

**🎙️ 整理拜访纪要（WF09）**：单击后上传纪要文件或粘贴文本，AI 抽取证据并结构化为拜访记录。需提供纪要内容。

**✅ 验证假设 & 更新现状（WF10）**：单击后 AI 对比"假设 vs 证据"，逐条输出验证结论并生成现状地图。需已有假设地图与拜访证据。

**🎯 营销地图角色卡（WF12）**：单击后 AI 生成或更新角色卡（客观层 + 主观层 + 行为分析）。需已有角色相关信息。

### 3.4 自然语言意图路由

不单击 chip 时，可直接在输入框用自然语言描述需求（如"帮我生成这个客户的业务地图"）。系统依次尝试 LLM 意图分类 → 关键词兜底 → 弹窗让你确认，自动路由到对应工作流；若都未命中则进入自由对话。

### 3.5 自由对话（Chat Mode）

不装填任何 chip 直接发送消息，即为自由对话模式，可用于通用问答、解释、优化、总结、头脑风暴。

### 3.6 文件上传与归档

**拖入文件**：将 PDF / DOCX / Excel / 文本文件直接拖入输入区，AI 自动解析、分类（财报 / 访谈纪要 / 系统清单 / 公开信息 / 待分类等 7 类）并评估覆盖度贡献。

**确认分类**：AI 给出分类建议后，单击"确认"完成归档，文件自动存入个人空间对应项目的资料子目录。

### 3.7 阅读 AI 回复

**思考过程**：AI 回复上方灰色折叠面板为推理链路，单击可展开查看，帮助判断结论可信度。

**来源证据**：回复中标注每条信息的出处（公开信息搜索结果、上传资料等）。

**候选结果卡**：当 AI 产出结构化数据（如一组角色卡、一版假设地图），以卡片形式展示，并附带"采纳 / 驳回"按钮。

### 3.8 产出确认（核心交互）

AI 产出的结构化数据不会直接入库，先进"候选区"由你确认：

**采纳**：单击候选卡的"采纳"，数据立即写入正式库，对应数据页面立即可见。

**Chat 调整**：不点采纳，直接在输入框用自然语言提出修改意见（如"L2 再拆细一点""角色卡漏了赵总"）。AI 基于原文加你的指令重新生成，并展示修改差异（diff），再次询问是否采纳。可多轮循环，直到满意。

**驳回**：单击"驳回"，产出进入偏差池 / 驳回池留痕，不污染正式数据。

### 3.9 审批提醒 Banner（Owner 专属）

当副手（Deputy）提交假设地图（WF07）产出待审核时，消息流上方 Banner 显示提醒。单击进入审核，可对比版本后选择"通过"（发布到业务地图）或"驳回 + 修改意见"（退回副手）。

### 3.10 角色消歧确认

当系统发现重名或高度相似的角色，Banner 弹出消歧确认，由你判定是同一人（合并）还是不同人（新建）。

### 3.11 标记为有价值

在自由对话中，单击某条 AI 回复的"标记为有价值"按钮，该内容自动存为个人空间对应项目目录下的 Markdown 知识片段。

## 四、业务地图

业务地图是独立数据页面，负责假设地图、现状地图、偏差池、前置分析、五维健康的查看与深度编辑。对话中采纳的地图数据会出现在此页面。

> 本页面不负责"让 AI 生成"——生成在对话页完成；此处只做查看、精修、版本管理。

### 页面总览

```
┌──────────────────────────────────────────────────────┐
│ 🗺️ 业务地图                                          │
│ 项目选择：[客户 / 项目 ▼]                             │
│ ──────────────────────────────────────────────────── │
│ L1: 5  L2: 18  L3: 42  L4: 15  已验证: 28/42 (67%)   │
│ [假设地图] [现状地图] [偏差池] [前置分析] [五维健康]   │
│ ──────────────────────────────────────────────────── │
│ 左侧：L1-L4 树形结构          右侧：节点详情面板      │
└──────────────────────────────────────────────────────┘
```

### 4.1 项目选择器与统计栏

**项目选择器**：页面顶部下拉切换项目，数据随之切换。

**统计栏**：显示当前项目 L1 / L2 / L3 / L4 各层节点数量与已验证比例。

### 4.2 假设地图视图（默认）

单击"假设地图"标签，展示 mapType = hypothesis 的 L1–L4 节点树，节点带置信度与来源类型标签。

**展开 / 折叠节点**：单击树形结构中的节点，展开或折叠其下层。

**查看节点详情**：单击任一节点，右侧详情面板按层级展示完整字段（L1 看 5 要素 + 五维健康；L2 看 8 要素；L3 看 11 要素含业务本体；L4 看能力达标率）。

**查看关联证据**：节点详情面板下方列出该节点关联的全部证据，单击可跳转拜访记录页对应证据。

### 4.3 现状地图视图

单击"现状地图"标签，展示 mapType = current 的节点。每个节点带"验证状态"标记（未验证 / 成立 / 部分成立 / 推翻 / 待补充），单击 linkedHypothesisId 可跳回原假设节点。

### 4.4 偏差池视图

单击"偏差池"标签，列出所有被"推翻"的节点，以"假设 vs 现状"逐条对比、红色高亮，用于复盘。

### 4.5 前置分析视图

单击"前置分析"标签，查看与编辑项目级的产业环境与战略定位分析，包含 5 个维度：行业价值链、客户行业地位、行业趋势与变化、客户战略定位、数字化驱动力。

### 4.6 五维健康视图

单击"五维健康"标签，查看 L1 公司级五维健康雷达图与 L2 / L3 逐域评分表。评分含 5 个维度：数字意识、数字神经、数字器官、数字血液、数字骨架。单击评分表某一行，跳转对应节点详情。

**重新评估**：单击"重新评估"按钮可手动触发五维健康重新计算。

**手动覆盖**：在节点详情面板可手动修改五维评分，修改后标注覆盖来源。

### 4.7 节点编辑

**手动新增节点**：单击"新增节点"，填写字段后直接写入正式库（用于补充 AI 未产出的节点）。

**编辑节点**：在节点详情面板单击"编辑"，修改字段后保存。

**删除节点**：在节点详情面板单击"删除"，移除该节点。

### 4.8 节点公开

在节点详情面板单击"标记为公开"，该地图片段在团队空间公开资产区可见（跨项目）。

### 4.9 版本管理（业务地图专属）

**查看版本列表**：单击"版本管理"，列出该项目历次采纳生成的版本快照。

**查看历史版本**：单击某个历史版本，查看其完整内容。

**版本对比**：选择历史版本与当前版本，查看两者差异（diff）。

**版本回滚**：单击"回滚"，用所选历史版本替换当前数据。

> 仅业务地图支持版本回溯，营销地图与拜访记录不做版本回滚。

## 五、营销地图

营销地图是独立数据页面，负责角色卡的查看与深度编辑。对话中采纳的角色卡会出现在此页面。

### 页面总览

```
┌──────────────────────────────────────────────────────┐
│ 🎯 营销地图                                          │
│ 项目选择：[客户 / 项目 ▼]                             │
│ 角色总数: 8  Champion: 2  最近更新: 2026-07-05        │
│ [组织架构][决策链][立场矩阵][采购时间线][角色卡][关系图][知识库]│
└──────────────────────────────────────────────────────┘
```

### 5.1 项目选择器与统计栏

**项目选择器**：页面顶部下拉切换项目。

**统计栏**：显示角色总数、Champion 数量、最近更新时间。

### 5.2 组织架构图视图

单击"组织架构"标签，以树形结构展示客户侧组织，标注汇报关系与关键岗位。数据来自角色卡的部门信息与角色关系。

### 5.3 决策链角色表视图

单击"决策链"标签，以表格展示各角色，含影响力条形图、综合评分与等级（Champion / 倾向我方 / 中立 / 反对），按影响力倒序排列。

### 5.4 角色-立场矩阵视图

单击"立场矩阵"标签，以散点气泡图展示：横轴为立场、纵轴为影响力、气泡大小为参与度。

### 5.5 采购流程时间线视图

单击"采购时间线"标签，展示五阶段通用模板：需求识别 → 方案评估 → 供应商筛选 → 商务谈判 → 合同签署。各阶段时间与节点由用户手动填写。

### 5.6 角色卡牌视图

单击"角色卡"标签，进入三栏界面：左侧角色列表、中间角色卡详情、右侧关联信息。中间角色卡含 5 个子标签页：

**客观信息**：查看与编辑角色的姓名、职位、部门、汇报对象、联系方式、角色类型、决策权、教育背景、过往经历、性格、沟通偏好、人际关系、与我方 / 竞品历史合作。相对稳定，数月更新一次。

**主观分析**：查看与编辑立场、显性 KPI、隐性个人诉求、对我方 / 竞品态度、参与度、影响力、支持度、综合评分（= 参与度 × 0.3 + 影响力 × 0.4 + 支持度 × 0.3）、等级、置信度、核心顾虑、影响杠杆。动态变化，每次接触后更新。

**行为分析**：以矩阵展示"观察到的行为 → AI / 人工解读 → 建议下一步动作"，每次接触后更新。

**态度历史**：以时间线展示立场变化记录，每条含日期、此前立场 → 当前立场、变化原因。新证据关联角色卡时由系统自动生成，可手动编辑或删除。

**话术库**：查看与编辑该角色关联的话术，及同类型角色的通用模板，按角色类型 × 场景组织。

### 5.7 角色关系图视图

单击"关系图"标签，展示交互式角色关系网络图：节点为角色卡，边为关系（汇报 / 影响 / 协作 / 对立 4 种类型，用不同颜色与线型区分）。单击节点跳转角色卡详情，悬停边显示关系描述。

**编辑关系**：可在关系图上手动新增、编辑、删除角色间关系。

### 5.8 知识库视图

单击"知识库"标签，查看跨客户通用的三类资料：五类角色识别速查表、行为分析速查表、新人培养四阶段流程。内容来源于方法论库，只读。

### 5.9 角色卡编辑

**手动新增角色卡**：单击"新增角色"，填写客观层与主观层字段后直接写入正式库。

**编辑角色卡**：在角色卡详情中单击"编辑"，修改字段后保存。

**删除角色卡**：在角色卡详情中单击"删除"，移除该角色卡。

### 5.10 角色卡公开

在角色卡详情中单击"标记为公开"，该角色卡在团队空间公开资产区可见（跨项目）。

### 5.11 话术管理

在角色卡"话术库"子标签页，可手动新增、编辑、删除话术，支持 Markdown 格式，按角色类型 × 场景组织。

## 六、拜访记录

拜访记录是独立数据页面，负责拜访记录时间线与证据的查看与筛选。对话中采纳的拜访纪要（需审批）会出现在此页面。

### 页面总览

```
┌──────────────────────────────────────────────────────┐
│ 📝 拜访记录                                          │
│ 项目选择：[客户 / 项目 ▼]                             │
│ 拜访次数: 12  涉及角色: 5                             │
│ ┌─ 拜访记录列表(倒序) ─┬── 证据筛选面板 ──────┐      │
│ │ 日期/类型/参与人/证据数│ 类型/强度/角色筛选    │      │
│ └──────────────────────┴──────────────────────┘      │
└──────────────────────────────────────────────────────┘
```

### 6.1 项目选择器与统计栏

**项目选择器**：页面顶部下拉切换项目。

**统计栏**：显示拜访次数与涉及角色数。

### 6.2 拜访记录列表

列表按时间倒序展示拜访记录卡片，每张卡片显示拜访日期、类型（现场访谈 / 电话沟通 / 视频会议 / 邮件 / 一句话记录）、参与人（我方 + 客户方）、地点、时长、拜访摘要、下一步行动、关联证据数 / 验证假设数 / 关联角色卡数。

**展开查看证据**：单击某条拜访记录，展开查看本次拜访产出的全部证据列表。

### 6.3 拜访记录新增（三种入口）

**对话触发 WF09**：在对话页单击"整理拜访纪要"，上传纪要或粘贴文本，AI 结构化后确认，即写入（推荐，质量最高）。

**页面手动新增**：单击拜访记录页"新增拜访"，手动填写字段后保存。

**对话随口提及**：在对话中说"今天见了王主任"，系统自动创建一条"一句话记录"。

> 拜访记录采纳需走审批流程，通过后才在本页面可见。

### 6.4 拜访记录编辑

单击某条拜访记录的"编辑"，可修改日期、类型、参与人、摘要、下一步行动等字段。

### 6.5 证据筛选面板

右侧证据筛选面板支持按三个维度筛选证据：

**按类型筛选**：客户原话 / 行为观察 / 角色态度信号 / 业务术语。

**按强度筛选**：强（多方交叉验证）/ 中（单一可靠来源）/ 弱（传闻或单次观察）。

**按角色筛选**：选择关联的角色卡。

筛选结果在下方列表实时展示，每条证据显示原始内容（引号体）、来源角色、关联假设。

### 6.6 证据关联与验证联动

**关联假设**：为某条证据指定关联的业务地图假设节点。

**确认建议验证状态**：系统根据证据内容与强度自动给出建议验证状态（如 3 条强证据支持 → 建议"成立"），单击确认后节点的验证状态自动更新。

**偏差池自动新增**：假设被验证为"推翻"时，偏差池自动新增一条偏差记录。

**现状地图自动复制**：现状地图数据从假设地图自动复制并标记验证状态，支持后续增删改查。

### 6.7 态度变化自动记录

当一条证据的来源关联到某角色卡且内容暗示态度变化时，系统自动在该角色卡的"态度历史"中新增一条记录（含日期、立场变化、原因）。

## 七、个人空间

个人空间是用户的私有文件存储区，按项目自动归档。

### 页面总览

```
个人空间 / <你的用户名> /
├── 数字化转型项目/  资料/{财报,访谈纪要,系统清单,公开信息,待分类} · 方案/ · 知识片段/
├── 信创迁移项目/  ...
└── 未关联项目/  （通用文件）
```

### 7.1 项目筛选

页面顶部"项目筛选框"下拉选择"全部项目"或指定项目，下方文件列表按所选项目过滤展示。

### 7.2 文件浏览

文件列表按目录结构展示，单击文件夹可展开 / 折叠，单击文件可预览或下载。

### 7.3 手动上传

单击"上传文件"按钮，选择本地文件上传。未关联项目的文件归入"未关联项目"目录。

### 7.4 自动归档目录

对话中上传并经 AI 分类的文件，自动存入对应项目的资料子目录（财报 / 访谈纪要 / 系统清单 / 公开信息 / 待分类）；WF06 产出的方案存入"方案"目录；"标记为有价值"的对话存入"知识片段"目录。

### 7.5 知识片段

"知识片段"目录存放由对话"标记为有价值"自动生成的 Markdown 文件，可打开查看与复用。

## 八、团队空间

团队空间是跨项目共享的知识资产区。

### 8.1 公开资产区

展示所有用户在各数据页面标记为"完全公开"的对象，按类型聚合分类（公开角色卡 / 公开业务地图片段 / 公开方案 / 公开知识片段）。单击可查看详情。

### 8.2 方法论库

展示管理员维护的方法论资产（Prompt 模板、画布 Schema、方法论规则），对普通用户只读。

### 8.3 对象公开与共享

在任意数据页面（业务地图 / 营销地图 / 拜访记录）的对象详情中：

**完全公开**：单击"标记为公开"，对象进入团队空间公开资产区，跨项目可见。

**指定用户共享**：单击"共享给"并选择用户，仅对方可见该对象。

> 默认项目外不可见（项目外全隔离）；要让项目外的人看到，必须显式公开或共享。

## 九、设置（管理员）

仅 admin 角色可见，单击导航栏"设置"进入。

### 9.1 用户管理

**待审批用户列表**：查看所有 pending_approval 状态的注册用户。

**通过审批**：单击某用户的"通过"，账号变为启用（active），可登录。

**拒绝审批**：单击"拒绝"，驳回该注册申请。

**手动创建用户**：单击"新增用户"，填写信息后创建，默认启用状态。

**启用 / 禁用账号**：在用户列表中切换账号状态为启用或禁用。

### 9.2 组织架构

系统使用自建三级组织架构（公司 → 部门 → 小组）。

**查看组织树**：以树形结构可视化展示三级组织。

**新增组织节点**：单击"新增"，填写名称、类型、上级、负责人、排序后保存。

**编辑 / 删除组织节点**：在节点上单击"编辑"或"删除"。

**批量导入**：单击"批量导入"，上传 CSV / Excel 文件，系统自动校验、去重并批量创建。

### 9.3 触发词配置

维护对话意图路由的关键词映射表，影响自然语言 → 工作流的识别准确度。可新增、编辑、删除触发词。

### 9.4 种子数据管理（方法论库）

维护团队空间方法论库的内容（Prompt 模板、画布 Schema、方法论规则），对普通用户只读。可新增、编辑、删除条目。

## 十、客户与项目管理

业务数据挂在项目下，项目挂在客户下。客户与项目可从对话页顶部项目选择器或专门的管理入口创建。

### 10.1 创建客户

单击"新建客户"，填写客户名称、行业、规模（大型 / 中型 / 小型）、区域、描述后保存。任何用户均可创建。

### 10.2 创建项目

单击"新建项目"，选择所属客户，填写项目名称、类型（诊断 / 试点 / 落地）、阶段、目标与描述后保存。系统自动为该项目创建专属 Agent（绑定 7 个 Skill + 3 个 Plugin），创建者自动成为 Owner。

### 10.3 项目成员管理（仅 Owner）

**按用户邀请**：选择系统用户加入项目，设为 Deputy。

**按部门授权**：选择组织部门授权，该部门所有成员自动获得项目访问权。

**移除成员**：将某成员移出项目。

### 10.4 项目设置（仅 Owner）

Owner 可修改项目的名称、类型、阶段、目标、描述等设置。

## 附录 B：术语与枚举值

**工作流（WF）**：WF02 资料上传 / WF03 缺口检查 / WF06 拜访方案 / WF07 假设地图 / WF09 拜访纪要 / WF10 验证假设 / WF12 角色卡。

**业务地图层级**：L1 价值链 / L2 业务域（含职能域、共性技术域）/ L3 细分场景 / L4 人才地图。

**五维健康**：数字意识 / 数字神经 / 数字器官 / 数字血液 / 数字骨架。

**地图类型**：hypothesis（假设）/ current（现状）。

**验证状态**：未验证 / 成立 / 部分成立 / 推翻 / 待补充。

**评审状态**：draft / pending_review / reviewed / rejected。

**拜访类型**：现场访谈 / 电话沟通 / 视频会议 / 邮件 / 一句话记录。

**证据类型**：客户原话 / 行为观察 / 角色态度信号 / 业务术语。

**证据强度**：强 / 中 / 弱。

**角色类型**：经济决策人 / 技术评估人 / 终端用户 / 教练支持者 / 采购财务。

**角色立场**：支持 / 中立 / 反对 / 观望。

**角色等级**：Champion（8–10）/ 倾向我方（5–7）/ 中立（3–4）/ 反对（1–2）。

**综合评分公式**：参与度 × 0.3 + 影响力 × 0.4 + 支持度 × 0.3。

**角色关系类型**：汇报（reports_to）/ 影响（influences）/ 协作（collaborates）/ 对立（opposes）。

**项目类型**：诊断 / 试点 / 落地。

**项目阶段**：线索筛查 / 拜访准备 / 现场验证 / 复盘。

**用户系统角色**：super / admin / user。

**用户状态**：待审批（pending_approval）/ 启用（active）/ 禁用（disabled）。

**组织类型**：公司 / 部门 / 小组。

**采购流程五阶段**：需求识别 → 方案评估 → 供应商筛选 → 商务谈判 → 合同签署。
