# 团队空间成员管理设计

## 目标

补齐团队空间成员管理能力：所有者可以删除成员、编辑成员权限、转让空间所有权；普通成员不能管理成员，但可以查看只读成员列表并主动离开团队。

## 权限规则

- 所有团队成员都可以查看成员列表。
- 只有空间所有者可以搜索待添加用户、添加成员、修改成员角色、删除成员、转让所有权。
- 普通成员不能看到成员管理操作，只看到成员列表和离开团队操作。
- 所有者不能直接离开团队，必须先把所有权转让给其他成员。
- 所有者不能删除自己，也不能把自己作为删除目标。
- 转让所有权的目标必须已经是该团队成员；转让后新所有者角色保证为 `editor`，原所有者继续保留成员身份。

## 接口设计

- `PATCH /api/team-spaces/{space_id}/members/{member_id}`：所有者修改成员角色。
- `DELETE /api/team-spaces/{space_id}/members/{member_id}`：所有者删除非所有者成员。
- `POST /api/team-spaces/{space_id}/owner`：所有者转让所有权给指定成员。
- `POST /api/team-spaces/{space_id}/leave`：当前成员主动离开团队，所有者调用时返回 400。

## 前端设计

团队空间详情页的成员弹窗保留现有成员列表。`space.is_owner === true` 时展示添加、权限编辑、删除和转让操作；普通成员只读成员列表，并展示离开团队按钮。

## 验证

后端新增 API 测试覆盖权限和状态变化；前端执行 `npm run build` 验证类型和打包。
